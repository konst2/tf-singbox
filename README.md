# tf-singbox

Учебный проект по развёртыванию sing-box на удалённом сервере с помощью Terraform без других инструментов.

## Как это работает

Клиент подключается к серверу c `sing-box` поверх TLS. На сервере трафик обрабатывает связка из двух контейнеров:
- `Caddy` по `TLS` проксирует запросы во второй контейнер.
- `sing-box` принимает очищенный трафик на внутреннем порту 1080 и отправляет его наружу (direct outbound настроен в конфиге sing-box).

Для посторонних запросов Caddy отдаёт статическую HTML-страницу-заглушку. TLS-сертификат Caddy получает автоматически через Let's Encrypt (функционал встроен в контейнер Caddy).

Всё это поднимается, по сути, одной командой `terraform apply`.

## Что делает Terraform

1. Заполняет шаблоны конфигов: серверный `Caddyfile`, серверный `config.json` для sing-box и клиентский `client.json`.
2. Подключается к серверу по SSH, создаёт каталог `/opt/sing-box/` и копирует туда docker-compose.yml, Caddyfile, конфиг sing-box и статическую заглушку сайта.
3. Запускает контейнеры через `docker compose up -d --force-recreate`.

После `apply` клиентский конфиг (`client_config/client.json`) можно перенести на устройство (мобильное или компьютер) и подключиться.

## Требования к серверу

- `Docker` и `docker-compose` установлены
- SSH-доступ по паролю (если по ключу -- переделать в `main.tf`)
- Открытые порты `80` (для получения сертификата с Let's Encrypt) и `443` (для использования sing-box)
- Домен направленный на IP сервера

Протестировано на `Ubuntu 24.04` с `Docker` и `docker-compose` из официального репозитория.

## Инструкция

Прежде всего требуется создать и заполнить `terraform.tfvars` по образцу из `terraform.tfvars.example`:

- IP, логин, пароль -- Данные для подключения на сервер через SSH.
- Логин и пароль для прокси --  Придумать. Они будут использоваться для подключения клиента.
- Домен и e-mail -- Домен должен смотреть на сервер. E-mail нужен для Let's Encrypt.

Далее, можно изменить `sing-box/site/index.html` -- это html статика которая отдаётся на несанкционированные запросы.

Затем выполнить:

```bash
terraform init
terraform plan
terraform apply
```

Клиентский конфиг появится в `client_config/client.json`.  
Его можно можно использовать в приложении sing-box на мобильном устройстве или компьютере.


После любых изменений следует запускать только:
```bash
terraform apply
```

Если нужно перезапустить без изменений:
```bash
terraform state rm null_resource.deploy
terraform apply
```

## Примечание

Для продакшена с сложным конфигурированием лучше взять Ansible.  
В рамках, определённых, текущими целями, только terraform достаточно.
